## /daily — механика

```
База: 10 (умножается на stabilizer.coeff)
Бонус стрика: [0, 10, 25, 35, 50, 60, 75]

Рандом-бонус поверх базы + стрика:
  +5   — 50%
  +25  — 35%
  +45  — 10%
  +85  — 4%
  +170 — 1%
```

- Streak увеличивается если `last_daily === вчера (UTC)`. Иначе сброс на 1.
- Повторный `/daily` в тот же день — одно сообщение, затем silent ignore.
- Сброс дня — по UTC (0:00).
- Коэффициент стабилизатора применяется к базе (10) и рандом-бонусу. Streak bonus не режется.

## /dice — кость

```
Команда: /dice N (создать игру со ставкой N)
Игра: инлайн-кнопки 🎲 Крутить / ✖️ x2 / ➗ x0.5 / 🏁 Закрыть
Ставка: N (фиксированная для всех игроков)
Механика: нажатие 🎲 Крутить → sendDice('🎲') → пауза 5с → результат
```

- dice value 1–6 определяет выплату:

| Значение | Множитель |
|----------|-----------|
| 6️⃣ | ×3 |
| 5️⃣ | ×2 |
| 4️⃣ | ×1 (возврат) |
| 1️⃣–3️⃣ | ×0 (проигрыш) |

- House edge: 0% (EV = 1.0) — как в `/coinflip`
- Dice-сообщение удаляется при следующем броске или закрытии
- Результат через 5 секунд (анимация кубика дольше, чем у слотов)
- При закрытии игры показывается итог: сколько создатель разбогател (или обеднел) за всю сессию

## /slots — слот-машина

```
Команда: /slots N (создать игру со ставкой N)
Игра: инлайн-кнопки 🎰 Крутить / ✖️ x2 / ➗ x0.5 / 🏁 Закрыть
Ставка: N (фиксированная для всех игроков)
Механика: нажатие 🎰 Крутить → sendDice('🎰') → пауза 2с → результат
```

- Telegram 🎰 dice кодирует 3 рила по 2-битному полю: `n = value - 1`, рилы = `[n&3, (n>>2)&3, (n>>4)&3]`.
  - `0` → 🎰 BAR
  - `1` → 🍇 Виноград
  - `2` → 🍋 Лимон
  - `3` → 7️⃣ Семёрка
- Выплаты определяются комбинацией:

| Комбинация | Множитель |
|-----------|-----------|
| 7️⃣ 7️⃣ 7️⃣ (value=64) | ×10 |
| 🎰 🎰 🎰 (value=1) | ×5 |
| 🍇 🍇 🍇 (value=22) | ×3 |
| 🍋 🍋 🍋 (value=43) | ×2 |
| Пара | ×1 (возврат ставки) |
| Всё разное | ×0 (проигрыш) |

- House edge: ~12.5% (EV = 0.875)
- Dice-сообщение удаляется при следующем спине или закрытии игры — не более одного dice на игру в чате
- Результат показывается через 2 секунды после отправки dice (пауза на анимацию)
- Игры хранятся в памяти (Map), теряются при рестарте
- При закрытии игры показывается итог: сколько создатель разбогател (или обеднел) за всю сессию

## /coinflip — мультиплеерная монетка

```
Команда: /coinflip N (создать игру со ставкой N)
Игра: инлайн-кнопки 🦅 Орёл / 🌿 Решка / ✖️ x2 / ➗ x0.5 / 🏁 Закрыть
Ставка: N (фиксированная для всех игроков)
Выигрыш (50%): N × 2 (округление вверх)
Проигрыш (50%): N сгорает
House edge: 0%
```

- Игра создаётся в сообщении с кнопками. Любой участник чата может нажать кнопку и сыграть.
- Ставка: минимум 10, максимум 500 бипок.
- `✖️ x2` / `➗ x0.5` — создатель игры может удвоить/половинить ставку (в пределах 10–500).
- `🏁 Закрыть` — только создатель может закрыть игру.
- После каждого хода сообщение обновляется: в лог (последние 5) добавляется запись с ✅/❌.
- Алерт показывает: выбор → результат → ВЫИГРАЛ/ПРОИГРАЛ + текущий баланс.
- Игры хранятся в памяти (`Map`, до 20 записей в логе), теряются при рестарте.
- При закрытии игры показывается итог: сколько все участники разбогатели (или обеднели) суммарно за всю сессию

## /transfer — комиссия

```
Отправитель платит: N
Получатель получает: N - ceil(N × 0.05)
Сгорает: ceil(N × 0.05)  (транзакция type='fee')
```

При N = 1 комиссия не взимается.

Комментарий: `/transfer @user 100 za pizza` — последнее число = сумма, всё после = комментарий.

## /work

```
Заработать: rand(5, 40) * stabilizer.coeff → Math.round → Math.max(1, ...)
  — базовый диапазон: 5–39
  — с учётом coeff: до 80 при coeff=2.0
Кулдаун: 2 часа
```

## /rain — распределение

```
Сумма: N
Сгорает: ceil(N × 0.05)  (транзакция type='fee')
Получают: 2–5 случайных участников чата (исключая отправителя)
Минимум: N >= 4 (после комиссии должно остаться ≥ 3 бипок)
Остаток делится случайными долями
```

Использует `bipbank.rainDistribute()` — атомарно: withdraw, deposits, fee burn в одной транзакции.

## /crack — взлом сейфа (ограбление банка)

```
Команда: /crack [N] (начать ограбление с опциональным залогом N)
Залог: 0–500 бипок, увеличивает шанс на +17% за каждые 100 бипок (макс +34%)
Лок: глобальный мьютекс на 60 секунд — одновременно грабить может только один
Сессия: 60 секунд на прохождение, после — автоотмена с возвратом залога
Кулдаун: 2 ч при провале, 24 ч при успехе

Механика — 3 этапа:
1. 🎲 Обход охраны: dice > (3 - floor(bet/100)) — успех при значении выше порога
2. 🎰 Взлом сейфа: слот-машина (три рила) — успех если есть пара или три одинаковых
3. 🦅/🌿 Замести следы: орёл/решка — угадал → 100% награды, мимо → 75%

Награда:
  stagesPassed = пройдено этапов (1–3)
  base = vault × (stagesPassed / 3)
  limit = 500 + floor(vault × 0.2)
  ceiling = max(MAX(balance FROM users WHERE user_id != грабитель), 2000)
  reward = min(floor(base), limit, ceiling)

В банк (vault) попадают:
  - залоги проваливших ограбление
  - 98% проигранных ставок в слоты/кости/монетку (2% сжигается)
При успешном ограблении 30% остатка vault сжигается (money sink).
```

## /ngbet — призовой пул NaganBet

```
Команда: /ngbet N (поставить N бипок в банк чата)
       : /ngbet (показать свою ставку и общий банк)
       : /ngbet cancel (вернуть все ставки в чате)
Механика: бот мониторит сообщения @naganbot и @itisdj_bot
  — при смерти игрока (матч по 13 паттернам) извлекается @username
    — если пользователь есть в bipki → pool.award: весь банк ему
    — если нет → дождь: 5% сгорает, остаток 3–5 случайным участникам
  — при ядерном взрыве (3 паттерна) все проиграли → дождь
  — если < 3 получателей для дождя — банк сгорает целиком (fee-транзакция)
  — пул очищается
Персистентность: ставки в game_pools, восстанавливаются при рестарте
```

Использует Pool API (`contribute` / `award` / `split` / `burn` / `refund`).

## Стабилизатор

Встроен в BipBank. Автоматически регулирует `/work` и `/daily`. Подробнее в [api.md](api.md#стабилизатор-экономики).

Ключевые особенности:
- **Защита от whales**: каждый пользователь учитывается не более 10000 бипок.
- **Диапазон**: коэффициент [0.1, 2.0] (было [0.5, 1.5]).
- **EMA-сглаживание**: коэффициент меняется плавно (фактор 0.3), исключая резкие скачки.

Прозрачно: `/economy` показывает текущий коэффициент.

## /charity — благотворительность (налог на богатство)

```
Команда: /charity (открыть меню)
Ставка: 1%–10% (обязательный минимум от баланса)
  — 0–999 бипок: мин. 1%
  — 1000–1999: мин. 2%
  — 2000–2999: мин. 3%
  — 3000–3999: мин. 4%
  — 4000+: мин. 5%
Сбор: раз в день, случайное время
  — платят пользователи с balance > average
  — сумма = floor(balance × rate / 100)

Буст дохода (применяется ко всем доходам):
  — rate=1%: 0% (нейтрально)
  — rate=5%: +20% к доходу
  — rate=10%: +45% к доходу

Забрать из банка (если balance < 100):
  — 25%/50%/90% от банка, раз в день
```

## /arena — Weekly Arena

```
Команда: /arena (показать топ недели)
Механика: автоматический зачёт net-выигрыша из coinflip/dice/slots
  — netDelta = payout − bet (только положительный)
  — счётчик сбрасывается каждый понедельник
  — лидер меняется — сообщение в чат
  — лидер держится 3+ дня — сообщение

Призы (в конце недели из vault):
  🥇 50% от min(vault, 500)
  🥈 30%
  🥉 20%

Никакой эмиссии — всё из vault (проигранные в игры бипки).
```
